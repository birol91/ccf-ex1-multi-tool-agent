Multi-Tool Support Agent — System Architecture

CCA-F Preparation Exercise 1 · manual agentic loop · claude-sonnet-4-6 · no framework

stop_reason — the only stop signal

Every Claude response says why it stopped. "tool_use" = "I need a tool, your turn" → loop continues. "end_turn" = "I'm done" → loop ends. Never parse the text to decide.

tool_use — a request, not an execution

A content block where Claude says "call this tool with these inputs" (carries id, name, input). Claude does not run it — you do, and return a tool_result matched by tool_use_id. The tools are the ones you define and code; Claude only picks from your list.

hook — deterministic guardrail

Code that runs before a tool executes. Gates every process_refund by tool name (100% deterministic): ≥ 500 → escalate to a human, < 500 → ask the user to confirm (yes/no). Even if the model mis-routes a status query to a refund, the user is asked first. Enforced in code, not the prompt — a prompt can be ignored, code cannot.
